# Underwater Anomaly Detection — PointNet++ Segmentation

A PyTorch-based pipeline for per-point semantic segmentation of man-made anomalies in synthetic underwater sonar/LiDAR point cloud scenes. The model identifies and localizes objects (cubes, spheres, cylinders) resting on a natural seabed using a **PointNet++ architecture with Feature Propagation** modules.

Developed as a capstone project for an uncrewed surface vessel (USV) anomaly detection system.

## Setup & Installation

> **Python 3.13** and an **NVIDIA GPU with CUDA 11.8** are required for full functionality.
> CPU-only training is possible but impractically slow for this dataset size.

### Option A — Conda (Recommended)

This is the fastest way to reproduce the exact environment used to train the model.
Requires [Miniconda](https://docs.conda.io/en/latest/miniconda.html) or Anaconda.

```bash
conda env create -f environment.yml
conda activate capstone
```

> **Note:** PyTorch Geometric packages (`torch-scatter`, `torch-cluster`, etc.) are installed
> via `pip` within the Conda env automatically. If Conda is unavailable, use Option B.

### Option B — pip (Manual)

**Step 1: Install PyTorch with CUDA support**

```bash
pip install torch==2.7.1+cu118 torchvision==0.22.1+cu118 torchaudio==2.7.1+cu118 \
    --index-url https://download.pytorch.org/whl/cu118
```

**Step 2: Install PyTorch Geometric C++ extensions**

These must exactly match your PyTorch + CUDA version:

```bash
pip install torch-scatter torch-sparse torch-cluster torch-spline-conv \
    --no-build-isolation \
    -f https://data.pyg.org/whl/torch-2.7.1+cu118.html
pip install torch-geometric==2.7.0
```

**Step 3: Install remaining dependencies**

```bash
pip install -r requirements.txt
```

### Verify GPU is detected

```bash
python -c "import torch; print(torch.cuda.get_device_name(0))"
```

## Full Workflow

### Step 1 — Generate Synthetic Data (requires Blender + BlAInder addon)

From the `data_generation/` directory, run the Blender-based scene generator. Each scene
randomly places one of three anomaly shapes (cube, sphere, cylinder) on a procedural seabed
and exports a labeled `.las` point cloud.

```bash
python generate_seabed.py \
    --count 1000 \
    --blender "C:\path\to\blender.exe" \
    --exact-balance
```

> **Crash recovery**: If interrupted, simply re-run the same command. The script detects
> already-generated scenes and resumes from where it left off.

### Step 2 — Inspect Labels (optional QA)

Visually verify that anomaly labels are correct before training:

```bash
python inspect_labels.py runs/<session_id>/output/scene_0.las
```

This outputs `scene_0_labeled.las` (color-coded by class) and `scene_0_anomaly_only.las`
(anomaly points only) which can be opened in CloudCompare or any LAS viewer.

### Step 3 — Prepare Dataset

Convert `.las` files to `.npy` arrays and split into `train/val/test`:

```bash
python split_and_prepare.py \
    --session runs/<session_id> \
    --out ../data
```

### Step 4 — Train

From the `src/` directory:

```bash
python train.py \
    --task segmentation \
    --model pointnet2seg \
    --data_dir ../data \
    --batch_size 8 \
    --epochs 100 \
    --class_weight 30.0
```

**To resume after a crash**, add the `--resume` flag:

```bash
python train.py --task segmentation --model pointnet2seg --data_dir ../data --resume
```

Checkpoints are saved to `../checkpoints/`:
- `pointnet2seg_segmentation_best.pth` — best validation mIoU (use this for inference)
- `pointnet2seg_segmentation_latest.pth` — last completed epoch (used for resuming)
- `pointnet2seg_segmentation_history.csv` — per-epoch metrics log

### Step 5 — Evaluate

```bash
python evaluate.py \
    --checkpoint ../checkpoints/pointnet2seg_segmentation_best.pth \
    --data_dir ../data \
    --export_viz 5
```

This prints per-class IoU, mIoU, and a confusion matrix to the terminal. With `--export_viz N`,
it also writes `N` color-coded Ground Truth vs Prediction `.ply` file pairs to
`src/evaluation_visuals/` for visual inspection.

### Step 6 — Plot Training Metrics

Open `visualize_results.ipynb` in VS Code or Jupyter and run all cells. It reads the
training history CSV and produces high-resolution plots saved to `results/`.

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| **Segmentation over Classification** | Per-point labeling provides spatial localization of anomalies, not just presence/absence. |
| **Weighted Cross-Entropy (weight=30)** | Anomaly points are <1% of each scene. Without upweighting, the model trivially predicts "terrain" everywhere. |
| **AMP (Automatic Mixed Precision)** | Halves VRAM usage on GTX 1060 6GB, enabling stable batch_size=8 without OOM errors. |
| **Gradient Clipping (max_norm=1.0)** | Prevents exploding gradients in early training epochs when loss landscape is steep. |
| **Domain Randomization** | Augmentations (Z-rotation, jitter, dropout, scale) prevent overfitting to clean synthetic data and simulate real sonar noise. |
| **Synthetic Data Pipeline** | 1,000 labeled scenes generated for $0 using Blender + BlAInder, versus prohibitive cost of real underwater acquisition. |
